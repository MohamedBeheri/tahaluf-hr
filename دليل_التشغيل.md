# نظام تقييم الأداء — التحالف الوطني
## دليل التشغيل والرفع (دينامِك + صلاحيات دخول)

التطبيق **ديناميكي بالكامل**: البيانات تُحفظ في قاعدة بيانات سحابية (**Firebase / Firestore**)،
وكل موظف يدخل بحسابه ويرى ما يسمح به دوره فقط. الاستضافة على **Netlify**.

---

## الأدوار والصلاحيات (٣ مستويات)

| الدور | يرى | يضيف / يعدّل / يحذف |
|------|------|--------|
| **مدير عام** | كل الإدارات والموظفين والتقييمات | كل شيء: الموظفون، الإدارات، الأقسام، البنود، **المستخدمون**، الاستيراد/التصدير، التقييم |
| **مدير إدارة** | موظفو إدارته فقط ونتائجهم | **موظفو إدارته** + تقييمهم + التقارير |
| **موظف** | بياناته الشخصية ونتيجة تقييمه فقط | اطلاع فقط (لا يضيف ولا يعدّل ولا يقيّم) |

> الحماية مطبّقة على مستويين: في الواجهة (إخفاء الصفحات/الأزرار)، **وفي قاعدة البيانات نفسها**
> عبر Firestore Security Rules — حتى لو حاول أحد تجاوز الواجهة، الخادم يمنعه.

---

## أولاً: تجربة فورية بدون أي إعداد (وضع المعاينة المحلي)
افتح `index.html` في المتصفح مباشرة. سيعمل التطبيق ببيانات تجريبية تُحفظ في متصفحك،
وتقدر تجرّب الأدوار الثلاثة:

- `admin` / `123456` ← مدير عام
- `ahmed` / `123456` ← مدير إدارة
- `khaled` / `123456` ← موظف

> هذا الوضع للتجربة فقط (البيانات لا تتشارك بين الأجهزة). للنسخة الحقيقية أكمل الخطوات التالية.

---

## ثانياً: تجهيز Firebase (مرة واحدة)

### 1) إنشاء المشروع
1. ادخل <https://console.firebase.google.com> وسجّل بحساب Google.
2. **Add project** ← اكتب اسم المشروع ← Continue (تقدر تعطّل Analytics) ← **Create project**.

### 2) تفعيل تسجيل الدخول بالبريد
**Build → Authentication → Get started → Sign-in method → Email/Password → Enable → Save**.
(لا حاجة لتفعيل تأكيد البريد — الدخول يعمل مباشرة.)

### 3) إنشاء قاعدة البيانات
**Build → Firestore Database → Create database** ← اختر **Production mode** ← اختر أقرب منطقة ← **Enable**.

### 4) لصق قواعد الحماية
في **Firestore Database → Rules**: امسح الموجود، والصق محتوى ملف
`firebase/firestore.rules` بالكامل، ثم **Publish**.

### 5) إنشاء أول «مدير عام»
1. **Authentication → Users → Add user** ← أدخل بريد وكلمة مرور المدير العام ← **Add user**.
2. انسخ **User UID** الظاهر بجوار المستخدم.
3. **Firestore Database → (+) Start collection** ← اسم المجموعة: `profiles` ← Next.
4. في **Document ID** الصق **الـ UID** المنسوخ، وأضف الحقول التالية:
   | الحقل | النوع | القيمة |
   |------|------|------|
   | `name` | string | المدير العام |
   | `email` | string | بريد المدير العام |
   | `role` | string | `general_manager` |
   ثم **Save**.

### 6) ربط التطبيق بالمشروع
1. **Project settings (⚙️) → General → Your apps →** اضغط أيقونة الويب **`</>`** ← سجّل التطبيق.
2. انسخ كائن `firebaseConfig` المعروض.
3. افتح ملف `config.js` والصق القيم بدل `YOUR_...`:

```js
window.FIREBASE_CONFIG = {
  apiKey: "AIza....",
  authDomain: "xxx.firebaseapp.com",
  projectId: "xxx",
  storageBucket: "xxx.appspot.com",
  messagingSenderId: "....",
  appId: "1:....:web:...."
};
```

بمجرد وضع القيم يتحوّل التطبيق تلقائياً للوضع السحابي الحقيقي.

> أول دخول للمدير العام يُنشئ بنود التقييم الافتراضية تلقائياً.

## ثالثاً: الرفع على Netlify (مجاني)

### الطريقة الأسهل (سحب وإفلات)
1. ادخل <https://app.netlify.com> ← سجّل حساباً.
2. **Add new site → Deploy manually**.
3. اسحب **مجلد `hr-app` بالكامل** وأفلته في الصفحة.
4. خلال ثوانٍ يظهر رابط التطبيق (مثل `https://اسمك.netlify.app`). هذا رابطك النهائي.

> أي تعديل لاحق: اسحب المجلد مرة أخرى لتحديث الموقع.

### بعد الرفع — اضبط Firebase
في Firebase ← **Authentication → Settings → Authorized domains → Add domain** ←
أضف نطاق Netlify (مثل `اسمك.netlify.app`) حتى يعمل تسجيل الدخول على الموقع المرفوع.

## رابعاً: إضافة الموظفين والمستخدمين (من داخل التطبيق)

1. ادخل بحساب **المدير العام**.
2. **الإدارات والأقسام**: أضف/عدّل/احذف الإدارات والأقسام (أو استوردها من **استيراد/تصدير**).
3. **الموظفون → إضافة موظف**: أضف الموظفين (أو استورد ملف Excel من القالب الجاهز).
   كل صف موظف فيه أزرار **تعديل** و**حذف**.
4. **المستخدمون والصلاحيات → مستخدم جديد**: أنشئ حساب دخول لكل شخص:
   - **مدير إدارة** → اختر الإدارة المخصصة له (يقدر بعدها يدير موظفي إدارته: إضافة/تعديل/حذف/تقييم).
   - **موظف** → اربطه بسجل الموظف الخاص به (ليرى بياناته فقط).
   - البريد + كلمة المرور هما بيانات دخوله.

> ملاحظات:
> - زر «حذف» في صفحة المستخدمين يلغي صلاحيات المستخدم (ملف الـ profile). لإلغاء حساب الدخول
>   نهائياً احذفه أيضاً من **Authentication → Users** في Firebase.
> - تغيير بريد/كلمة مرور مستخدم قائم يتم من لوحة Firebase Authentication.
